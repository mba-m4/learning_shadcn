# draw.io 埋め込みトラブル対応ランブック

## 対象
- 対象画面: `/document/templates`
- 対象コンポーネント: `src/components/template-editor/drawio-editor-simple.tsx`
- 対象ドメイン: `embed.diagrams.net`（`app.diagrams.net` は iframe 埋め込み不可）

## 事象（今回）
- 「開くが編集UIが出ない」
- 画面上は `ファイル -> 終了` と `ヘルプ` のみ
- 図形ライブラリ・ツールバー・編集キャンバスが期待通り表示されない

## 直接原因（技術）
1. `embed=1` の意味を誤って扱った
   - `embed=1` は「親アプリとメッセージ連携する前提」のモード。
   - URLだけで期待する完全編集UIが常に出るわけではない。
2. 初期 `load` メッセージ未送信
   - `ready/init` 後に `{action: 'load', xml: ...}` を送らないと、編集状態に遷移しないケースがある。
3. URLパラメータの試行が先行し、プロトコル前提の検証が後回しになった
   - `ui=...`, `#R`, `#U` などの調整だけで解決する前提が誤りだった。
4. CSP 制約の見落とし
   - `app.diagrams.net` は `frame-ancestors` 制約で埋め込み不可。

## 間接原因（プロセス）
1. 失敗モードの切り分け不足
   - 「読み込み失敗」「CSPブロック」「embedプロトコル未完了」を同時に扱ってしまい、原因が混線した。
2. 仮説の優先順位が逆
   - 正しい順序は「仕様前提確認 -> 最小実装で再現 -> 追加パラメータ」
   - 実際は「パラメータ調整 -> UI差分確認 -> 仕様再確認」になっていた。
3. 受け入れ基準（DoD）の不足
   - 「編集可能」の定義（左ライブラリ表示、図形ドラッグ可、保存イベント受信可）が明文化されていなかった。
4. ログ戦略の不足
   - `ready/init` 受信、`load` 送信、`save/autosave` 受信の3点ログを最初から固定化していなかった。

## 何を調査しても解けなかった理由
- 症状は「URLの問題」に見えるが、実際は「埋め込みプロトコルの未完了」が根因だった。
- つまり、観測点（見た目UI）と真因（メッセージハンドシェイク）が異なる層にあり、
  パラメータだけ変えても本質的に改善しなかった。

## 有効だった考え方
1. 単一責務で検証する
   - まず「テンプレート連携を外す」
   - 次に「iframe表示のみ」
   - 最後に「ready/init -> load」の最小ハンドシェイクを追加
2. 仕様に戻る
   - draw.io embed mode FAQ の JSON protocol を基準にした
3. 観測可能性を作る
   - 受信イベントと送信アクションを必ずログで確認

## 恒久対策（再発防止）

### 1) 実装テンプレートを固定化
以下を埋め込み実装の最小雛形として固定する。

- URL: `https://embed.diagrams.net/?embed=1&proto=json&ui=kennedy&libraries=1&spin=0&saveAndExit=1&noExitBtn=0`
- `window.message` で `ready/init` を受信
- 1回だけ `{action:'load', xml: EMPTY_XML, autosave:1, saveAndExit:1}` を送信

### 2) 受け入れ基準（DoD）を明文化
次の4つを満たしたら「編集可能」と判定する。
1. 左の図形ライブラリが表示される
2. キャンバスに図形をドラッグできる
3. `save` または `autosave` イベントが受信される
4. リロード後も初期表示で編集UIが再現される

### 3) デバッグ手順を標準化
1. CSPエラー確認（`app.diagrams.net` を使っていないか）
2. 受信イベント確認（`ready/init`）
3. `load` 送信確認
4. `save/autosave` 受信確認
5. ここまで成立後にのみURLパラメータ微調整

### 4) 禁止事項（今回の学び）
- `embed=1` 使用時に、`load` 未実装のまま UI パラメータだけ調整して進めない
- `app.diagrams.net` を iframe 埋め込み先として使わない
- 保存不具合と表示不具合を同時に直そうとしない（必ず分離）

## 初動チェックリスト（運用）
- [ ] 埋め込み先ドメインが `embed.diagrams.net`
- [ ] URLに `embed=1&proto=json`
- [ ] `message` 受信ログがある
- [ ] `ready/init` 後に `load` を送っている
- [ ] `load` のXMLが空でない
- [ ] `save/autosave` の受信ログがある
- [ ] A4（`pageWidth=827`, `pageHeight=1169`）で初期化される

## 今回の是正内容（実施済み）
- `drawio-editor-simple.tsx` に `ready/init -> load` の最小ハンドシェイクを実装
- 空のA4 XMLを初期ロード
- `embed` モード用URLに統一

## 補足
本件は「実装スキル不足」よりも「仕様前提の順序ミス」と「検証戦略不足」が主因。
再発防止の本質は、URL調整より先にプロトコル成立を検証する運用へ切り替えること。
